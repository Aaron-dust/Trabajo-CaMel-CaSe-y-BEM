Dos realidades en programacion: Conociendo mejor al CaMel CaSe y code BEM.

Dentro de la programación existen muchos tipos de formatos en los cuales un programador puede escribir su código. Hoy hablaremos acerca de Camel Case y del código BEM, los cuales ayudan al usuario en su práctica de programación. Desglosaremos sus ventajas y desventajas, así como la manera de usarlos de la forma más eficiente posible.

¿Que es el CODE BEM?

BEM (Block Element Modifier) es una metodología revolucionaria para nombrar clases en CSS que transforma el caos del código desordenado en una estructura clara, predecible y fácil de mantener. No es un lenguaje, sino un estilo inteligente que permite a los desarrolladores crear interfaces limpias y modulares, donde cada parte tiene su lugar y su nombre bien definido. ¡Con BEM, el diseño y la programación van de la mano para que tu código hable un idioma que todos puedan entender!

Ventajas de usar BEM.

  1. Mejora la legibilidad del código. Los nombres estructurados facilitan entender rápidamente qué representa cada clase.
  2. Facilita el mantenimiento y la escalabilidad. Al ser un sistema organizado, es más sencillo modificar y ampliar el código sin causar conflictos.
  3. Evita conflictos de nombres CSS. Gracias a su nomenclatura específica, reduce la posibilidad de que diferentes estilos se sobreescriban accidentalmente.
  4. Promueve la reutilización de componentes. Los bloques y elementos pueden usarse en diferentes partes del proyecto sin problemas.
  5. Facilita el trabajo en equipo. Al seguir un estándar común, todos los desarrolladores entienden y siguen la misma lógica.
  6. Permite estilos más específicos y claros. Los modificadores permiten definir variaciones sin necesidad de crear nuevas clases complejas.

¿Como se ve esto en la practica BEM?.

  • <.div class="producto"> <.h3 class="producto__titulo">Zapatos<./h3> <.p class="producto__precio">$49.99<./p> <./div>
    • ➤ Ventaja: Facilita identificar claramente a qué bloque pertenecen el título y el precio.
  • <.div class="tarjeta tarjeta--destacada"> <.p class="tarjeta__descripcion">Oferta limitada<./p> <./div>
    • ➤ Ventaja: Permite modificar el diseño de una tarjeta específica sin cambiar las demás.
  • <.form class="formulario"> <.input class="formulario__input" type="text" placeholder="Tu nombre"> <.button class="formulario__boton">Enviar<./button> <./form>
    • ➤ Ventaja: Se mantiene una estructura ordenada y clara para el formulario y sus elementos.
  • <.ul class="lista"> <.li class="lista__item lista__item--seleccionado">Elemento 1<./li> <.li class="lista__item">Elemento 2<./li> <./ul>
    • ➤ Ventaja: Es fácil resaltar o modificar solo los elementos seleccionados de la lista.

CamelCase, algo mas que un animal para la programacion.

.
Camel case es una convención de escritura utilizada en programación y desarrollo de software para nombrar variables, funciones o identificadores. Consiste en unir varias palabras sin espacios y capitalizar la primera letra de cada palabra excepto la primera. Esta práctica facilita la lectura y comprensión del código, haciendo que los nombres compuestos sean más claros y fáciles de distinguir visualmente.

Situaciones practicas para el uso de CamelCase.

  1. Nombres de variables Ayuda a identificar fácilmente el propósito de la variable al separar visualmente las palabras, mejorando la legibilidad.
  2. Nombres de funciones o métodos Facilita distinguir rápidamente qué hace la función, haciendo el código más intuitivo y fácil de entender.
  3. Identificadores en objetos y clases Permite mantener una nomenclatura clara y consistente que facilita la navegación y manipulación de objetos complejos.
  4. Parámetros de funciones Mejora la claridad en la definición de funciones, ayudando a entender qué datos se esperan recibir.
  5. Propiedades dentro de objetos JSON Favorece la coherencia y compatibilidad en la comunicación entre sistemas y APIs, evitando errores de interpretación.

Demostracion visual del uso bueno del CamelCase.

  • <.div id="contenedorPrincipal">Contenido aquí<./div>
    • ➤ Ventaja: Usar camelCase en el id ayuda a identificar claramente el contenedor principal y facilita la manipulación vía JavaScript.
  • <.button onclick="procesarDatosUsuario()">Enviar<./button>
    • ➤ Ventaja: Nombrar funciones con camelCase hace que el código sea más legible y fácil de entender, mejorando el mantenimiento.
  • <.input type="text" name="nombreUsuario" placeholder="Ingresa tu nombre">
    • ➤ Ventaja: Los nombres de atributos en camelCase facilitan la conexión clara entre el HTML y el código JavaScript que los manipula.
  • <.p class="mensajeErrorValidacion">Por favor, corrige los errores<./p>
    • ➤ Ventaja: Clases en camelCase ayudan a mantener un estilo coherente y facilitan su uso en CSS y JavaScript.
